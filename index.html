Driver Database

Skills:

  • Java
  • JUnit Unit Testing
  • Data Ecapsulation

The Driver Database program is a "code sample" of a driver database which stores drivers in a tree structure and their tracking information regarding recent trips. This program had an emphasis on abstraction, reusability, and testability. Unit tests were implemented with the JUnit framework as build time dependencies gaurunteeing only clean builds are created. Test results were automatically logged in a human readable format for reliability and debugging.

Full program analysis of run times, memory usage and strucuture avaliable in README.

Preternship

This project-internship was a 2 person project where we developed statistical pattern recognition software in C++ to analyze network logs, diagnose system performance and predict incidents of high network stress to prevent system outages for a representative at IBM. We had to identify critical tasks, determine project dependencies and design considerations, develop task effort and duration estimates with a partner and present deliverables to our representative. We delivered weekly code reviews and professional memorandums detailing progress, assessed risks and alternatives, developed prototypes, and determined new objectives using the Spiral Software Development Model.

Chess Server

This Chess interface was created in Java featuring remote access for players to connect to chess games over WiFi. I utilized client/server connection modules to allow users to host their own games or use a third-party server if needed. Server sided error checking allows for only valid inputs from both of the users during the game.
